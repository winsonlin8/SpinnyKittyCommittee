# MSPM0G3507 I2C Target Firmware

This CCS project is the peripheral controller for the Spinning Cat Speaker.
It receives playback control commands from the ESP32 over I2C and drives the
DC motor and 35-pixel WS2812 LED ring in sync with the music.

---

## How It Works

The firmware runs as an I2C target at address **0x48**.

On receipt of **CMD_PLAY (0x01)** the main loop begins advancing through two
pre-computed animation tables at 20 ms / frame:

- **Motor** — software PWM on PA22 drives the L298N motor driver.
  Speed data comes from `music_data.h` (6 735 samples generated by offline
  beat/onset/energy analysis of the song).

- **LEDs** — bit-banged WS2812 protocol on PA15 drives the 35-pixel ring.
  Animation parameters come from `led_beat_params.h` (5 800 frames: comet
  head position, brightness, tail length, and colour index).

On receipt of **CMD_PAUSE (0x02)** the loop halts and all outputs hold their
last state.

Both data tables are generated by
`Audio Characterization/spinny_kitty_spinning_testing.py` from an offline
audio analysis of the target song.

## Peripherals & Pin Assignments

| Peripheral | Pin   | Function                            |
|:-----------|:------|:------------------------------------|
| I2C0       | PA0   | SDA — I2C data line (from ESP32)    |
| I2C0       | PA1   | SCL — I2C clock line (from ESP32)   |
| GPIOA      | PA15  | WS2812 LED ring data line           |
| GPIOA      | PA25  | L298N IN1 — motor direction A       |
| GPIOA      | PA24  | L298N IN2 — motor direction B       |
| GPIOA      | PA22  | L298N ENA — motor speed (SW PWM)    |
| DEBUGSS    | PA20  | SWCLK (debug only)                  |
| DEBUGSS    | PA19  | SWDIO (debug only)                  |

## Source Files

| File                  | Description |
|:----------------------|:------------|
| `i2c_target.c`        | Main firmware: I2C target ISR, motor PWM, WS2812 bit-bang |
| `music_data.h`        | Pre-computed motor PWM table (6 735 values × 20 ms) |
| `led_beat_params.h`   | Pre-computed LED animation table (5 800 frames × 4 bytes) |
| `i2c_target.syscfg`   | SysConfig peripheral configuration |
| `ti/comm_modules/i2c/target/i2c_comm_target.c` | TI I2C comm module |

## Building

Open this folder as a CCS project. The project targets the **MSPM0G3507** device.
Use **Run → Debug** to build, flash, and start a debug session.

---

## LaunchPad Jumper Settings

| Jumper        | Setting | Purpose |
|:--------------|:--------|:--------|
| J101 13:14    | ON      | Connect PA19 to XDS-110 SWDIO for debugging |
| J101 15:16    | ON      | Connect PA20 to XDS-110 SWCLK for debugging |
| J19 1:2       | ON      | Optional: on-board pull-up for PA0 (SDA) |
| J20 1:2       | ON      | Optional: on-board pull-up for PA1 (SCL) |

> External 2.2 kΩ pull-ups to 3.3V on SDA and SCL are recommended over the
> on-board pull-ups when connecting to the ESP32.

## Device Migration

This project targets the **MSPM0G3507** (LP-MSPM0G3507 LaunchPad). To migrate
to another MSPM0 device, follow the
[CCS SysConfig migration guide](https://software-dl.ti.com/msp430/esd/MSPM0-SDK/latest/docs/english/tools/ccs_ide_guide/doc_guide/doc_guide-srcs/ccs_ide_guide.html#sysconfig-project-migration).
